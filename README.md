AI-Based Income Protection for Delivery Partners

PROBLEM :

Delivery partners working with platforms like Zomato and Swiggy often lose a part of their income due to situations like heavy rain, extreme heat, pollution, or sudden restrictions. These are things they cannot control, but they directly affect their daily earnings.

At present, there is no simple system that helps them recover this lost income.

---

OUR IDEA :

We are proposing a simple insurance-based solution that protects delivery workers from income loss caused by such external conditions.

The main idea is to automatically detect these situations and provide compensation without requiring the worker to manually apply for a claim.

---

Who This Is For

- Food delivery partners
- People who depend on daily or weekly earnings
- Workers affected by weather or area-based disruptions

---

HOW IT WORKS ?

Step 1: Registration

The user signs up and provides basic details such as location and average daily income.

Step 2: Weekly Plan

Instead of a monthly system, the user pays a small weekly amount (around ₹20–₹50). This matches their earning pattern.

Step 3: Risk Calculation

The system uses past data (like weather patterns in that area) to estimate how risky the location is and adjusts the weekly amount accordingly.

Step 4: Event Detection

If a situation like heavy rainfall or extreme temperature happens, the system identifies it automatically using data.

Step 5: Claim Process

The claim is triggered automatically. The user does not need to apply manually.

Step 6: Payment

The calculated compensation is sent to the user through a simulated payment method.

---

Fraud Prevention

To avoid misuse:

- Location is verified
- Duplicate claims are checked
- Basic activity validation is done

---

Technology Used

- Frontend: HTML, Tailwind CSS
- Backend: Django / Node.js
- Database: SQLite or MySQL
- External Data: Weather API (or sample data)

---

What the System Includes

- A simple login and registration system
- A dashboard showing coverage and claims
- Automatic claim triggering
- Basic admin view for monitoring
